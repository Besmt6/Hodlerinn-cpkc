<analysis>**original_problem_statement:**
Build a two-part application for a hotel named Hodler Inn.

**Guest Portal Requirements:**
1.  **Unified Check-In:** A single Check-In flow that handles both new and returning guests.
2.  **Employee Validation:** Guests can only register/check-in if their Employee ID exists in a pre-approved list managed by the admin. The guest's name is non-editable and pulled from this list.
3.  **Check-Out:** Guests check out by entering their room number.
4.  **Voice & Visuals:** The portal provides voice guidance and visual feedback, including highlighted input fields to guide the user.
5.  **Help:** A How to Use section with a video tutorial.

**Admin Backend Requirements:**
1.  **Dashboard:** A detailed report dashboard showing all guest activities.
2.  **Billing & Reporting:** Calculate stays based on calendar nights (not hours) and provide Excel, PNG, and PDF reports. The admin can edit records from the billing report.
3.  **Employee Management:** Admins can add, edit, delete, and bulk-import an approved list of employees.
4.  **AI Data Collection Agent:** An AI agent that can log into a third-party railroad portal () and scrape historical sign-in reports to build the master employee list.
5.  **AI Verification Agent:** An AI agent that runs daily to sync with the railroad portal, verifying stays and marking No Bill for guests who appear on the portal's list but did not stay at the hotel.
6.  **Telegram Notifications:** Send alerts to a group chat for new check-ins, especially flagging first-time check-ins for admin review.

**User's preferred language**: English
